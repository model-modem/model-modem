    GitHub Pages Custom Domain⚓︎

    If you are deploying your MKDocs site via the mkdocs gh-deploy command and finding that it is resetting your custom domain name settings, there is a solution for that.

    There is an undocumented feature of MKDocs that it will publish your site to GitHub Pages using a custom domain name only if you create a file called CNAME which contains the custom domain name of your site. If you do not create this special file, your site's custom domain name will be wiped out each time you deploy your site.

    For example if your site was called modelmodem.com you would (and I did) create a file called CNAME and place it at the root of your docs_dir.

    modelmodem.com

    File /docs/CNAME
